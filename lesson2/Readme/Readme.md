## Урок 2. Неймспейсы и изоляция

Контейнеризация, как концепция, и технологии изоляции, используемые для реализации контейнеров, имеют глубокие корни в мире Linux и технологиях ядра операционной системы.

Идея контейнеризации начала своё развитие в Unix-подобных системах ещё в конце прошлого века, когда были введены концепции виртуальной машины (chroot) и пространства имёен пользователя (user namespaces). Они предоставляли базовую изоляцию процессов и файловой системы, позволяя создавать относительно изолированные окружения.

Однако, реализация контейнеризации в полной мере стала возможной лишь с развитием технологий ядра Linux, таких как control groups (cgroups) и namespaces (пространства имён - перевод, далее NS). Эти функции стали стандартом для создания изолированных окружений и управления ресурсами. Об этом и поговорим в этом уроке.

В контексте контейнеризации, NS представляет собой механизм операционной системы, который обеспечивает изоляцию различных системных ресурсов, путем предоставления каждому контейнеру своего собственного видимого пространства для этих ресурсов. Каждый контейнер видит только свои собственные ресурсы, такие как процессы, файловые системы, сетевые интерфейсы, и другие системные атрибуты, отделённые от других контейнеров и хоста. NS обеспечивают высокую степень изоляции, безопасности и контроля над ресурсами, создавая виртуальные среды, где контейнеры могут работать как автономные сущности, независимо друг от друга.

Когда контейнер запускается, используя технологию контейнеризации, такую как Docker, Docker внутренне использует NS в Linux для изоляции процессов, сетевых интерфейсов, файловых систем и других ресурсов. Это предотвращает возможность вмешательства контейнера в другие контейнеры и обеспечивает безопасность и изоляцию. Каждый контейнер видит только свои собственные ресурсы, что позволяет им работать как миниатюрные автономные системы.

Таким образом, NS в контейнеризации позволяют контейнерам предоставить изолированные окружения, эффективно управлять ресурсами, обеспечивать безопасность и облегчать развёртывание приложений в независимых и надёжных средах.

Ниже рассмотрим типы NS с примерами их применения в Docker:

#### 1. PID Namespace (изоляция процессов)

PID namespace предоставляет изолированное пространство идентификаторов процессов для каждого контейнера. 

**Пример:**

```
docker run -d --name container1 ubuntu sleep 1000
docker run -d --name container2 ubuntu sleep 1000

```

В этом примере мы создаем два контейнера, каждый из которых выполняет команду "sleep", приостанавливающую выполнение процесса на указанное количество секунд. Поскольку каждый контейнер находится в своем собственном PID namespace, процессы внутри каждого контейнера не пересекаются с процессами в других контейнерах или на хосте.

#### 2. Mount Namespace (изоляция файловой системы)

Mount namespace создает изолированное представление файловой системы для каждого контейнера. 
**Пример:**

```
docker run -d --name container1 -v /hostdir1:/containerdir1 alpine sleep 1000
docker run -d --name container2 -v /hostdir2:/containerdir2 alpine sleep 1000

```
В данном случае каждый контейнер видит свою собственную корневую файловую систему, отделенную от файловых систем других контейнеров и хоста.

 
#### 3. Network Namespace (изоляция сетевых ресурсов)

Network namespace изолирует сетевые ресурсы, включая сетевые интерфейсы, таблицы маршрутизации, firewall правила и другие параметры сети.

**Пример:**

```
docker network create isolated_net
docker run -d --name container1 --network=isolated_net nginx
docker run -d --name container2 --network=isolated_net nginx

```
В этом примере мы создаем изолированную сеть под названием isolated_net с помощью команды docker network create. Затем мы запускаем два контейнера (container1 и container2), указывая им использовать эту изолированную сеть.

#### 4. IPC Namespace (изоляция межпроцессного взаимодействия)

IPC namespace изолирует межпроцессное взаимодействие, такое как очереди сообщений и семафоры, что позволяет контейнерам работать в изолированных средах без конфликтов взаимодействия.


#### 5. UTS Namespace (изоляция параметров системы)

UTS namespace изолирует два параметра: название хоста и доменное имя. 

**Пример:**

```

docker run --rm -it --name container1 --hostname=container1 ubuntu
docker run --rm -it --name container2 --hostname=container2 ubuntu

```

В данном примере каждый контейнер имеет своё уникальное название хоста и доменное имя, полностью отделенные друг от друга.

#### 6. User Namespace (изоляция идентификаторов пользователей)

User namespace позволяет изолировать идентификаторы пользователей и групп внутри контейнера, обеспечивая запуск процессов с привилегиями пользователя, безопасно изолированными от основной системы.

**Пример:**


docker run --rm -it --name container1 --user=1000:1000 ubuntu


Каждый из этих неймспейсов обеспечивает важные механизмы изоляции, позволяя контейнерам работать как независимые сущности с высокой степенью безопасности.
