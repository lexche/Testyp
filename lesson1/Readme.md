# Урок 1. Контейнеризация

В этом уроке мы изучим такое понятие как контейнеризация. Что это? Откуда она появилась? Зачем это нужно? Почему об этом столько разговоров? Начнём.

 Контейнеризация в ИТ представляет собой способ создания изолированных областей для запуска и управления приложениями и их зависимостями. Эти области, называемые контейнерами, позволяют приложениям работать как отдельным сущностям, но при этом использовать общее ядро операционной системы. Это значительно снижает нагрузку на системные ресурсы и делает развертывание, управление и масштабирование приложений более эффективными.

Иными словами, контейнеризация в IT подобна созданию набора отдельных "комнат" для приложений внутри компьютера. Эти "комнаты" позволяют запускать разные приложения в изолированных процессах, где каждое приложение имеет свой собственный "приватный" уголок для работы. Например, представь, что у тебя есть множество небольших квартир в одном здании, где каждая квартира представляет отдельное приложение.

Традиционная виртуализация подобна созданию полноценных домов (виртуальных машин) для каждого приложения, что может быть недостаточно эффективно, особенно если нужно запустить много небольших приложений, каждое из которых использует мало ресурсов.

В контрасте с этим, контейнеризация предоставляет возможность создавать эти "квартиры" для приложений, которые могут разделить общие ресурсы, такие как системное ядро. Это делает процесс более эффективным и управляемым.

 Таким образом, контейнеризация позволяет развертывать и управлять приложениями внутри отдельных изолированных "комнат", где они работают независимо друг от друга, но в то же время могут совместно использовать общие ресурсы, что делает процесс развертывания и управления более гибким и масштабируемым.

 ##### Цель иллюстрации: продемонстрировать, что при виртуализации каждое приложение - это дом, при контейнеризации - квартира. При подобной ассоциации расчитываю, что появится понимание: на обслуживание квартиры понадобиться меньше ресурсов, чем на дом. Квартиры в виде контейнеров специально, чтобы продемонстрировать возможность их транспортировки. Можно каждый дом и квартиру подписать: Приложение 1, Приложение 2 и т.д.


 ### Квиз:  Выбери о чём идёт речь виртуализация или контейнеризация.

 1.  Этот метод изоляции использует общее ядро операционной системы для запуска приложений.

   - Правильный ответ: Контейнеризация. Общее ядро операционной системы позволяет контейнерам обмениваться ядром вместо создания отдельных ядер для каждой виртуальной машины.

2.  Этот подход позволяет запускать несколько изолированных экземпляров операционной системы на одном физическом сервере.

   - Правильный ответ: Виртуализация. Благодаря виртуализации можно создавать несколько виртуальных машин, каждая из которых работает под управлением своей собственной операционной системы.

3.  Использует гипервизоры для создания и управления виртуальными машинами.

   - Правильный ответ: Виртуализация. Гипервизоры используются для управления виртуальными машинами.

4.  Этот метод обеспечивает быструю загрузку и запуск, а также облегчает использование ресурсов благодаря общей основе операционной системы.

   - Правильный ответ: Контейнеризация. Контейнеры обычно имеют быстрое время запуска и используют общие ресурсы хост-системы, что способствует оптимальному использованию ресурсов.
